1 What is an IPU?

An IPU (Intelligence Processing Unit) is an AI accelerator from Graphcore, designed from scratch for machine intelligence. It targets fine-grained parallelism — thousands of cores each doing their own work on data held in on-chip memory.

A GPU runs the same op across cores (SIMT); an IPU runs different ops — MIMD.

The main idea

Keep the whole model on-chip, and let every core run independently — avoiding slow external-memory trips and handling sparse, irregular work that stalls a GPU.

2 Where IPUs came from

The IPU maps AI's computational graphs directly onto hardware — which is exactly where the chip gets its name.

Graphcore argued that AI models are really computational graphs full of irregular, sparse operations — a poor fit for the GPU's lock-step model.

Analogy: a huge open-plan office

A GPU is an army marching in perfect step. An IPU is a huge office of thousands of workers, each with their own desk and notebook (local SRAM), free to work on different tasks — then everyone pauses to swap notes.

3 Tiles & the BSP cycle

The IPU is built from thousands of tiles — each a small core with its own local SRAM, free to run a different program.

6 Applications

The IPU shines on irregular, sparse, graph-shaped AI:

🕸️ Graph neural networks 🧠 Sparse / fine-grained models 🔬 Research AI 💹 Low-latency finance 🧫 Molecular modeling

✅ Why pick an IPU

  • Huge on-chip memory bandwidth
  • Independent cores for irregular work
  • Strong on non-dense-matmul models
  • Clean graph programming (Poplar)

⛔ Not ideal for

  • Peak dense matmul → GPU/TPU
  • Broadest ecosystem (CUDA wins)
  • Already GPU-tuned workloads

8 IPU in one paragraph

The IPU is Graphcore's AI processor built around thousands of independent (MIMD) cores and huge on-chip SRAM, executing in a clean Bulk Synchronous Parallel cycle. By keeping entire models on-chip and letting cores run their own programs, it targets fine-grained and sparse AI workloads that don't map neatly onto a GPU's lock-step matrix engines — technically distinctive, though challenged by CUDA's dominance and now backed by SoftBank.

Interesting when…

  • Your model is sparse / graph-like
  • You want on-chip memory bandwidth
  • You want independent per-core execution

Future trends

  • SoftBank-backed development
  • Deeper bets on sparsity
  • 3D wafer-on-wafer stacking
  • Lessons for AI-silicon startups
