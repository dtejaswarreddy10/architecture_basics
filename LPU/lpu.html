Introduction

What is an LPU?

An LPU (Language Processing Unit) from Groq runs large language models with extremely low, predictable latency — generating tokens at record speed. Its architecture is the TSP (Tensor Streaming Processor). A GPU decides at runtime what each core does; an LPU decides at compile time — everything is pre-planned.

Where did it come from?

Groq (2016, founded by Jonathan Ross, who helped create Google's TPU) targeted the biggest pain in serving LLMs: inconsistent latency. GPUs are fast on average but jittery because caches and schedulers make timing variable. Groq's answer: remove all nondeterminism.

The main idea

Make the hardware perfectly predictable by letting software schedule everything — no caches, no branch prediction, no dynamic scheduling. The compiler maps exactly which op runs on which unit at which cycle.

Summary

LPU in one paragraph

The LPU is Groq's AI inference processor, built on a Tensor Streaming Processor that removes caches, branch prediction, and dynamic scheduling so the compiler can plan every operation cycle-by-cycle. This makes timing fully deterministic, delivering the lowest, most predictable latency and record token-generation speeds for LLMs — specialized for inference, not training.

Choose an LPU when…

  • You serve LLMs needing fast replies
  • Latency is critical
  • You want deterministic performance
  • You're in the inference phase

Future trends

  • The "inference era" suits the LPU
  • Ever-higher tokens-per-second
  • Deterministic large-scale systems
  • Fast, cheap cloud inference
