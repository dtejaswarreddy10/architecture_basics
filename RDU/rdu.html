Introduction

What is an RDU?

An RDU (Reconfigurable Dataflow Unit) from SambaNova Systems is built on a Reconfigurable Dataflow Architecture (RDA). Instead of running instructions on fixed cores like a CPU/GPU, the RDU rewires its compute and memory units to form a custom pipeline that mirrors the model's dataflow graph.

Where did it come from?

SambaNova (2017, founded by Stanford professors and veteran chip leaders) saw that AI models are dataflow graphs, yet GPUs execute them as endless kernel launches with constant memory traffic. The RDU maps the whole graph onto hardware and keeps data moving.

The main idea

Configure the hardware to the model, not the model to the hardware. A compiler arranges the RDU's units into the precise pipeline a model needs, so intermediate results stream between stages on-chip instead of bouncing to memory.

Basic Functionality & Architecture

Simple analogy: a custom factory line per product

A GPU is a general workshop where every part is carried back and forth to the same machines. The RDU builds a dedicated assembly line shaped like your model — each station feeds the next, so the product flows straight through.

Applications

🧠 Large language models 📚 Long-context / huge-parameter models 🏢 Enterprise & government AI 🔀 Composition of Experts 🔬 On-prem / regulated AI

✅ Why pick an RDU

  • Big models on fewer chips (large memory)
  • Less memory overhead via dataflow
  • Flexible — reconfigures per model
  • Delivered as a turnkey system

⛔ Not for

  • General computing → CPU
  • Graphics → GPU
  • The broadest open ecosystem (CUDA)
  • Tiny edge / low-power → NPU

Summary

RDU in one paragraph

The RDU is SambaNova's AI processor built on a Reconfigurable Dataflow Architecture: a compiler arranges its Pattern Compute Units and Pattern Memory Units into a spatial pipeline shaped like the model's dataflow graph, so data streams between stages on-chip with little memory overhead. With large memory capacity it runs very large models on fewer chips, delivered as turnkey enterprise systems — distinct from the GPU's instruction-stream model.

Choose an RDU when…

  • You run very large models / long context
  • You want big models on fewer chips
  • You prefer a turnkey on-prem system
  • You benefit from dataflow efficiency

Future trends

  • Large on-package memory for frontier models
  • Composition-of-Experts serving
  • Enterprise & sovereign AI appliances
  • Dataflow architectures maturing
